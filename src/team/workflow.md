# 工作流程

## 项目流程

### 1. 想法收集
- **渠道**：微信群、论坛置顶帖
- **格式**：简单说明做什么、为什么做、谁来用
- **收集人**：F组

### 2. 会议讨论
- **组织**：F组（策划部）负责安排
- **时间**：每周一次，线上会议
- **内容**：
  1. 新提案初筛
  2. 进行中项目进度同步
  3. 遇到的问题协调

### 3. 开发实施
```
认领任务 → 开发测试 → 内部演示 → 收集反馈 → 优化发布
```
- **任务板**：GitHub Projects 或看板工具
- **代码管理**：分支开发，PR合并
- **测试**：至少一位其他成员验证

## 协作规范

### 沟通方式
- **日常**：微信群即时沟通
- **文档**：项目README必填
- **周报**：简要说明本周完成、下周计划

### 资源申请
需要设计素材、服务器资源等，直接联系E组（资源部）。

### 发布宣传
项目完成后，由C组（宣传部）制作介绍内容并发布。
